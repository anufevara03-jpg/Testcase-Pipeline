# SCEM项目商户端激活码管理（激活码列表）模块说明

## 一、 页面概览

- **统计卡片**：展示“激活码总数”、“启用激活码数”、“停用激活码数”、“激活码分组数”四个核心指标。
- **筛选条件**：支持按“激活码”（下拉选择）、“平台”（WhatsApp、Telegram、Line）、“状态”（全部、启用、停用）、“创建时间”（时间段选择器）进行筛选。筛选条件支持清空和重置。
- **操作按钮**：包含“添加激活码”、“批量移动分组”、“批量删除”。
- **列表展示**：左侧为分组导航，右侧为激活码列表（支持多选）。列表展示激活码(备注)、最近登录时间、社交平台、在线端口、分配端口、使用字符数、重复线索范围、多设备登录、聊天备份、创建时间、状态及操作等信息。

## 二、 顶部核心统计

### 1. 指标与计算口径

| 指标             | 计算口径                               |
| :--------------- | :------------------------------------- |
| **激活码总数**   | 当前商户下激活码总数量                 |
| **启用激活码数** | 状态为启用的激活码总数量               |
| **停用激活码数** | 状态为停用的激活码总数量               |
| **激活码分组数** | 当前商户下账号分组总数量，默认分组计入 |

### 2. 后台与PC联动及注意事项

- **生成规则**：激活码是新建成功后由系统自动生成的唯一码，用于登录PC客户端。生成规则为12位随机码，由大小写字母+数字组成。
- **唯一性**：激活码至少同一商户下必须唯一。
- **分组**：激活码关联的平台决定PC登录后可管理哪些平台账号；激活码关联的分组用于后台管理。

### 3. 激活码去重范围影响

- **影响说明**：激活码去重范围仅影响非首次登录进入的线索，不影响首次上报/手动上报/线索清零/工单重复。
- **范围**：激活码去重范围为账号下全部工单时，会和底粉管理中的全部底粉进行去重统计。

**影响详情：**
| 动作 | 是否受激活码去重范围影响 | 是否导入底粉 | 结果 |
| :--- | :--- | :--- | :--- |
| 后续真实进粉的象粉/废粉判断 | 是 | 取决于激活码去重范围是否含底库 | 决定线索为新/重复线索 |
| 首次登录已有联系人上报 | 否 | 不导入底粉封控 | 进入平台工单底粉，不形成线索 |
| 线索清零/工单重复 | 否 | 不导入底粉封控 | 把当前线索沉淀为平台工单底粉 |

### 4. 激活码去重范围逻辑与示例

- **含底库**：底粉管理里的导入底粉数据 + 对应去重范围内的平台工单底粉数据。
- **不含底库 = 只查线索**；含底库 = 查线索 + 对应范围内的底粉。
- **限制**：仅在同平台内去重，跨社交平台数据相互隔离，不参与互相去重。

**各去重范围含义及示例：**
| 去重范围 | 含义 | 示例 |
| :--- | :--- | :--- |
| **当前工单** | 只在当前平台工单内查重，查的是当前工单已产生的线索，不查底粉 | WhatsApp 工单里账号 A 已产生 @aa 线索，账号 B 再添加 @aa，算重复；如果 @aa 不属于账号 A 的线索，只在当前工单底粉里，账号 B 添加时，仍可算新增 |
| **当前工单（含底库）** | 在当前平台工单内查重，同时查当前范围底库：导入底粉 + 当前平台工单底粉 | @aa 已在当前工单底粉里，账号 B 再添加 @aa，算重复 |
| **账号下全部工单（含底库）** | 1. 当前商户下，当前社交平台的全部平台工单 + 同平台底库。<br>2. 跨社交平台数据相互隔离，不参与互相去重。 | 激活码A和激活码B为同一个后台账号创建，激活码A下的TG社交账号1加了@Binance，为新增，再用激活码B（去重范围账号下全部工单）下的TG社交账号2加了@Binance，为重复 |
| **关联账号下全部工单（含底库）** | 在当前后台账号及其关联后台账号下的所有平台工单内查重，同时查关联范围内底库。 | 后台账号 A 和后台账号 B 做了关联，A 的工单已有 @aa，B 的工单再进 @aa，算重复 |
| **激活码分组内** | 在当前激活码所属分组内查重，范围是该分组下激活码产生的工单 | 激活码 X、Y 在同一分组，X 的工单已有 @aa，Y 的工单再进 @aa，算重复；其他分组不参与 |
| **指定激活码去重** | 只在指定的激活码集合内查重 | 指定 X、Y 两个激活码共享去重，X 已有 @aa，Y 再进 @aa 算重复；Z 不参与 |

## 三、 分组设置

### 1. 分组定位

- **管理方式**：按分组管理激活码。
- **展示逻辑**：点击分组后，右侧列表只展示该分组下的激活码；点击全部展示全部激活码。
- **默认分组**：默认分组为系统分组，不允许删除；删除其他分组时，其下激活码移动到默认分组。

### 2. 新增/编辑分组

| 字段         | 类型     | 是否必填 | 校验/说明                                                            |
| :----------- | :------- | :------- | :------------------------------------------------------------------- |
| **分组名称** | 文本输入 | 是       | 最长20个字符；同一商户下分组名称不可重复，重复提示“分组名称已存在”。 |

### 3. 删除分组

- 删除前需二次确认：“确认删除该分组吗？删除后该分组下的激活码将移动到默认分组。”
- 删除分组不删除激活码，若去重范围为激活码分组内，后续新增/补录线索按移动后的分组范围去重，历史已统计线索默认不回溯重算。

## 四、 新增/编辑激活码配置（所有功能字段）

### 1. 端口分配

- **类型**：单选，是必填项。
- **枚举值**：动态分配、固定分配。默认选择动态分配。
- **动态分配说明**：PC 登录时把账户剩余未使用的端口，动态分配给未自动绑定的激活码。动态分配可用数 = 端口总数 - 固定分配的总数 - 动态分配类型下在线社交账号的总数。
- **固定分配说明**：表示该激活码最多只能占用指定端口数；固定分配需大于0。

### 2. 分组

- **类型**：下拉选择，是必填项。默认选择默认分组。

### 3. 账号权限

- **类型**：复选框，否，不需要必填。
- **枚举值**：支持移除账号、支持线索分析、支持工单日志、支持线索清零、支持工单分流。默认选择支持移除账号。
- **说明**：控制外部打开分享的工单链接时，能/不能移除账号、看线索分析、看工单日志、线索清零、工单分流。

### 4. 社交平台

- **类型**：复选框，是必填项。
- **枚举值**：支持 WhatsApp、Telegram、Line，至少选择一个。默认全选。
- **PC限制**：PC 登录后只展示和允许登录使用所选平台。
- **状态关联**：正常状态下，勾选已开通的平台，工单状态为启用；禁用的状态下，未勾选未开通的平台，工单状态为禁用（禁用后历史数据不删除）。

### 5. 线索去重范围

- **类型**：单选，是必填项。
- **枚举值**：当前工单、当前工单(含底库)、账号下全部工单(含底库)、关联账号下全部工单(含底库)、激活码分组内、指定激活码去重。默认选择“当前工单”。
- **说明**：PC 新增线索时按该范围判断重复，影响线索列表、平台工单、结算统计。

### 6. 激活码工作时间

- **类型**：单选 + 时间段，是必填项。
- **枚举值**：暂不设置、固定时间。默认选择暂不设置。
- **说明**：暂不设置表示全天可用；选择固定时间时，需要填写工作开始和结束时间。
- **影响**：创建分流链接且分流模式选择「自主选择主号」时，会显示社交账号对应激活码的工作时间（示例图显示为 09:00 - 18:00）。

### 7. 自动回复

- **类型**：复选框，否，不需要必填。
- **枚举值**：关键词回复、欢迎语回复。
- **说明**：勾选后，PC 端在对应平台、激活码、状态满足时触发。

### 8. 多设备登录

- **类型**：单选，是必填项。
- **枚举值**：开启、关闭。默认选择关闭。
- **说明**：开启后同一激活码允许多台 PC 同时登录；关闭后激活码在新设备登录时需要将旧设备挤下线，提示“此激活码已在新设备登录”。

### 9. 聊天备份

- **类型**：单选，是必填项。
- **枚举值**：开启、关闭。默认选择开启。
- **说明**：开启后 PC 会话记录同步进入聊天记录相关页面。

### 10. 客户画像共享

- **类型**：单选，是必填项。
- **枚举值**：共享、不共享。默认选择不共享。
- **说明**：
  - 同一商户、同一社交平台、同一个联系人，开启共享的激活码可以看到和使用彼此共享的客户画像信息。
  - 共享切换为不共享时，原有的私有客户画像会隐藏，可见共享画像数据；切换回共享后可见原有的私有画像。
  - 私有客户画像：某个激活码在“不共享”状态下填写的客户画像信息，只属于这个激活码自己，属于私有信息。
  - 共享画像数据：激活码开启“共享”后，在 PC 客户画像里填写或使用的数据，是当前联系人的共享客户画像。其他同平台+同联系人，且也开启共享的激活码，就能看到和使用这份共享客户信息。

### 11. 数据脱敏

- **类型**：单选，是必填项。
- **枚举值**：开启、关闭。默认选择关闭。
- **说明**：开启后 PC 端不显示联系人的手机号、用户名、用户ID等敏感数据。

### 12. 置零时间

- **类型**：时间选择器，是必填项。默认选择00:00。
- **说明**：用于设置统计置零时间点，影响平台工单和线索统计的日结算清零口径。

### 13. 数据权限

- **类型**：下拉选择。
- **说明**：控制当前激活码相关数据可以被哪些部门下的员工看到。

### 14. 备注

- **类型**：多行文本，否，不需要必填。
- **说明**：最多50字，列宽展示在激活码备注位置，用于内部标识。

## 五、 激活码管理操作

### 1. 保存时校验

- 必填项不能为空，否则提示：“请输入xxx，请选择xxx”。
- 保存成功后关闭抽屉，刷新当前列表；PC 同步时使用最新配置。
- 激活码是新建成功后由系统自动生成的唯一码，用于登录PC客户端，生成规则为12位随机码，由大小写字母+数字组成。
- 激活码至少同一商户下必须唯一。

### 2. 启用/停用激活码

| 操作           | 提交结果                                                                                                                                                                                                         |
| :------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **启用激活码** | 选中激活码状态变为启用，可用于 PC 客户端登录。                                                                                                                                                                   |
| **停用激活码** | 1. PC 客户端不可再使用该激活码新登录，如激活码已登录PC客户端，将被强制下线，提示“该激活码已停用或删除”。<br>2. 系统自动回收已占用的端口数。<br>3. 激活码对应的所有工单的状态设置为禁用，禁用状态无法再新增线索。 |

### 3. 批量移动分组

- 批量移动分组时，至少选择1条激活码；未选择时按钮禁用，鼠标悬浮时提示：“至少选中一项数据”。
- **字段**：目标分组（下拉选择，必填项）。只能选择已有分组。

### 4. 登录日志

- 点击查看当前激活码的登录详情，内容包括：激活码、平台、登录时间、登录设备、登录 IP、登录地区、登录结果。
- **用途**：用于排查 PC 登录失败、异地登录、多设备冲突、端口占用等问题。

### 5. 分享工单

- 同平台工单页面-分享工单功能。
- 分享页面权限由新增/编辑激活码中的分享页面权限控制；若未授权，分享页面不展示对应功能。

### 6. PC 登录场景说明

| 场景               | 后台判断                                                             | PC 结果                                                                                                 |
| :----------------- | :------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------ |
| **首次登录**       | 激活码存在、状态启用、平台已配置、端口有余额、当前时间符合工作时间。 | 登录成功后加载平台、数据权限、脱敏、自动回复、聊天备份等配置，并写入最近登录时间和登录日志。            |
| **激活码不存在**   | 输入的激活码未匹配后台记录。                                         | 登录失败，提示“激活码不存在”。                                                                          |
| **停用激活码**     | 状态为停用。                                                         | PC 客户端不可再使用该激活码新登录，如激活码已登录 PC 客户端，将被强制下线，提示“该激活码已停用或删除”。 |
| **端口不足**       | 在线端口达到分配端口上限。                                           | 不可继续新增 PC 端口或登录新激活码，提示“当前端口数已达上限，请提升端口数或升级套餐”。                  |
| **多设备登录关闭** | 同一激活码已有效设备在线。                                           | 关闭后该激活码在新设备登录时需要将旧设备挤下线，提示“此激活码已在新设备登录”。                          |

### 1. 查询条件

| 字段/操作    | 类型         | 说明                                                                |
| :----------- | :----------- | :------------------------------------------------------------------ |
| **激活码**   | 下拉选择     | 按激活码筛选。                                                      |
| **平台**     | 下拉选择     | 支持 WhatsApp、Telegram、Line。筛选后只展示包含该平台权限的激活码。 |
| **状态**     | 下拉选择     | 支持按全部、启用、停用筛选。停用激活码不可在 PC 客户端登录。        |
| **创建时间** | 时间段选择器 | 按激活码创建时间范围筛选，开始时间和结束时间均可清空。              |

| 列区域           | 规则                                                                                                                                                                                          |
| :--------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **勾选列**       | 1. 支持当前页多选，用于批量启用、停用、移动分组。<br>2. 未勾选时批量按钮禁用且鼠标悬浮时提示“至少选中一项数据”。                                                                              |
| **激活码(备注)** | 展示激活码编号和备注，激活码用于 PC 客户端登录。                                                                                                                                              |
| **最近登录时间** | 展示该激活码最近一次 PC 登录时间，暂无则显示 “-”。                                                                                                                                            |
| **社交平台**     | 展示该激活码可登录的平台，多个平台以图标和数量展示。                                                                                                                                          |
| **账号分组**     | 展示激活码归属分组。                                                                                                                                                                          |
| **在线端口**     | 1. 当前登录在线的社交账号数量（如 WhatsApp、Telegram等）。<br>2. PC 登录或打开更多账号时需要校验端口余量，超出后不可继续新增登录端口。                                                        |
| **分配端口**     | 展示动态分配、固定分配。                                                                                                                                                                      |
| **使用字符数**   | 展示翻译或会话相关的字符消耗统计。                                                                                                                                                            |
| **重复线索范围** | 1. 展示当前工单、当前工单(含底库)、账号下全部工单(含底库)、关联账号下全部工单(含底库)、激活码分组内、指定激活码去重。<br>2. PC 新增线索时按该范围判断重复，影响线索列表、平台工单、结算统计。 |
| **多设备登录**   | 1. 展示开启/关闭开关。<br>2. 关闭时同一激活码只允许一个设备；开启时允许多设备同时登录。                                                                                                       |
| **聊天备份**     | 1. 展示开启/关闭开关。<br>2. 开启后 PC 会话记录进入聊天记录页面；关闭时不做备份。                                                                                                             |
| **创建时间**     | 展示激活码创建时间。                                                                                                                                                                          |
| **状态**         | 1. 展示启用/停用开关。<br>2. 启用时登录 PC；停用后无法登录 PC，且原登录会强制下线，回收端口，并禁用对应工单。<br>3. 停用后该激活码将 PC 客户端退出登录，并提示“该激活码已停用或删除”。        |
| **操作**         | 1. 激活码启用时：查看、编辑、分享、移动分组、聊天记录。<br>2. 激活码停用时：移动分组、删除。                                                                                                  |
