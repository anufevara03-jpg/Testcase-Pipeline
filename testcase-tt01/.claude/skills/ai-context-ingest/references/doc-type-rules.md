# 文档类型提取策略

> 各类源文档的识别特征、端/模块路由与提取要点。本文件是 `SKILL.md §3/§4` 的展开。

## 通用规则

- 每条提取的事实必挂来源 `（源：{文件名} §{章节}）`。
- 高危内容（字段类型/枚举/数值/错误码/测试数据）逐项溯源；缺来源写 `待补充` + 注明需提供什么。
- 推测性内容标 `（未证实）`，置于"冲突/待确认"区，不混入正文。
- 文档覆盖多端/多模块时按端/模块拆分，分别产出提取报告。

---

## 1. 数据库设计文档

**识别特征**：含 `CREATE TABLE`、表清单总览、ER 图、字段定义、设计约定。

**端/模块路由**（按域）：
- A 域（平台管理：三方平台/套餐/翻译线路/商家/CMS）→ web-admin 模块 + shared 公共字典
- B 域（商家后台：激活码/账号设备/代理/客户/线索/社群/运营/组织权限）→ web-merchant 模块
- C 域（客户端接口支撑：登录日志/会话端口/API Key）→ pc-client 模块
- 设计约定、公共字段、ER 概览、跨表关键说明 → `shared/common-data-model.md`
- 一张表被多端引用时：以"维护方"端为主写入 data-model，其余端引用 `（见 shared/common-data-model.md §X）`

**提取要点**：
- 实体字段表：字段 / 类型 / 约束 / 说明（类型为高危，逐字段溯源）
- 状态流转：状态枚举与触发操作（枚举为高危）
- ER 关系：实体间基数（1:N / N:M）与关系说明
- 设计约定：引擎/字符集/主键/公共字段/金额精度/时间戳/命名/租户隔离等（数值精度为高危）
- 关键设计说明：如端口按会话计、去重底库、计费闭环等

**高危项**：字段类型（BIGINT/VARCHAR(64)/DECIMAL(12,4) 等）、枚举值、数值精度、索引唯一约束。

## 2. 模块需求说明

**识别特征**：标题含"模块说明"，内容描述页面概览/统计/筛选/字段配置/操作/查询条件/列表规则。

**端/模块路由**：
- 端关键词：商户端/商家后台 → web-merchant；管理端/管理后台/平台运营 → web-admin；客户端/桌面/PC 端 → pc-client
- 模块名：取标题模块名转 kebab-case（"激活码管理" → activation-code-management）
- 端或模块不明确 → 列候选问用户

**提取要点**（按 5 文件维度）：
- **requirement.md**：用户故事、验收标准（`- [ ]` 列表）、业务规则、约束依赖
- **api.md**：接口名（若有）；参数/响应/错误码通常缺失 → `待补充`
- **data-model.md**：模块涉及的实体字段，**列类型回查数据库设计文档**（双来源：模块文档语义 + DB 文档类型）
- **ui-spec.md**：页面布局、跳转逻辑、前端校验规则（数值/字符长度为高危）、异常状态提示文案（文案为高危）
- **test-notes.md**：已知缺陷（无则"暂无记录"）、回归重点、测试数据（通常 `待补充`）

**高危项**：校验数值（字符长度、端口数）、提示文案原文、枚举（去重范围/状态/分配类型）、字段类型（回查 DB 文档）。

---

## 3. 项目描述

**识别特征**：标题含"项目描述/项目背景"，描述端定位、技术栈、业务流程、模块架构。

**端/模块路由**：
- 桌面应用/Electron → pc-client；web 商家后台 → web-merchant；管理后台 → web-admin
- 写入对应端 `platform-overview.md`

**提取要点**：
- 端简介（端名称/类型/目标用户/定位）
- 技术栈（前端框架/UI 库/状态管理）
- 交互范式
- 核心模块职责与事务边界
- 端到端业务流程（可保留 mermaid 图）
- 与其他端的关系（共享规则/接口/数据，指向 shared/；跨端流程指向 cross-platform/）

**高危项**：具体数值（如版本号、延迟毫秒数）、技术栈名称——逐项溯源，文档未明确写 `待补充`。

---

## 4. 接口文档

**识别特征**：含接口路径、请求方法、请求参数表、响应示例、错误码表。

**端/模块路由**：需用户指明归属端/模块；无提示则按接口名匹配现有模块的 api.md，匹配不到问用户。

**提取要点**：
- 路径、请求方法、鉴权方式
- 请求参数表（字段/类型/必填/说明）—— 类型为高危
- 成功响应示例
- 错误码表（错误码/含义）—— 每个错误码为高危

**高危项**：参数字段类型、错误码、必填约束、数值范围。

---

## 5. 其他

**识别特征**：不属于上述任一类型。

**处理**：列出文档中可识别的结构化信息与疑似归属端/模块，问用户确认类型与去向后再提取；不得强行套用上述策略。
