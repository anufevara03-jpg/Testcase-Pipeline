# 提取报告模板（ingest → maintainer 契约）

> `ai-context-ingest` 产出的提取报告按本模板组织，在对话中直接呈现（不落盘），交由 `ai-context-maintainer` 据此写入。
> maintainer 读取本报告后：按目标文件分组写入对应端/模块（模板填充）、执行铁律 B 写前闸门、完整性检查、更新 changelog。

## 模板

```markdown
# 提取报告：{源文件名}

- **源文件**：{相对路径}
- **文档类型**：{数据库设计文档 / 模块需求说明 / 项目描述 / 接口文档 / 其他}
- **目标端/模块**：{端名}/{模块名}（或 shared/{文件}；或 {端}/platform-overview）
- **端/模块置信度**：{高 / 中 / 低}（低则附候选与确认结果）
- **提取日期**：{YYYY-MM-DD}

## 一、提取内容（按目标文件分组）

### requirement.md
- **用户故事**：
  - 作为 {角色}，我希望 {做什么}，以便 {效果}。
- **验收标准**：
  - [ ] 场景：{场景}，预期：{结果}（源：{文件} §{章节}）
- **业务规则**：
  1. {规则}（源：{文件} §{章节}）
- **约束与依赖**：{依赖项}

### api.md
- **接口名** (METHOD /path)
  - 描述：{...}（源：{文件} §{章节}）
  - 鉴权：{... 或 待补充}
  - 请求参数：{表格 或 待补充}
  - 响应示例：{... 或 待补充}
  - 错误码：{表格 或 待补充}

### data-model.md
- **实体关系**：{A} 1---N {B}（源：{文件} §{章节}）
- **核心实体字段**：

  | 字段 | 类型 | 约束 | 说明 | 来源 |
  | ---- | ---- | ---- | ---- | ---- |
  | {字段} | {类型} | {约束} | {说明} | {文件} §{章节} |

- **状态流转**：{状态A} --{操作}--> {状态B}（源：{文件} §{章节}）

### ui-spec.md
- **页面列表/布局**：{...}（源：{文件} §{章节}）
- **页面跳转逻辑**：{...}（源：{文件} §{章节}）
- **前端校验规则**：

  | 字段 | 规则 | 触发时机 | 错误提示 | 来源 |
  | ---- | ---- | -------- | -------- | ---- |

- **异常状态展示**：{...}（源：{文件} §{章节}）

### test-notes.md
- **已知缺陷**：{记录 或 "暂无记录"}
- **回归测试重点**：{...}（源：{文件} §{章节}）
- **测试数据说明**：{... 或 待补充}

## 二、高危内容清单

> 铁律 A 范围，逐项溯源；状态=已溯源 / 待补充。

| 类别 | 内容 | 来源 | 状态 |
| ---- | ---- | ---- | ---- |
| 字段类型 | {字段} {类型} | {文件} §{章节} | 已溯源 |
| 状态枚举 | {枚举项} | {文件} §{章节} | 已溯源 |
| 具体数值 | {数值} | {文件} §{章节} | 已溯源 |
| 错误码 | — | — | 待补充 |
| 测试数据 | — | — | 待补充 |

## 三、冲突/待确认清单

> 与现有 ai-context/ 比对发现的冲突或推测，编号 C1/C2…；不裁决，等用户确认。

- **C1**：{冲突描述}（方A来源：{...} / 方B来源：{...}）—— 建议处理：{（未证实）标注并向用户确认}
- **待确认**：{推测/歧义项}（未证实）

## 四、写前闸门自检（铁律 B）

1. 本批高危内容是否每项都挂了有效来源？{是 / 否→列项}
2. 有无高危项被编造填充？{无 / 有→改为待补充}
3. 推测内容是否已标（未证实）并隔离？{是 / 否→调整}

## 五、写入计划

- **新建/更新**：{端}/{模块}/{文件}（列具体文件）
- **跨端影响**：{若有，列出受影响端/模块}
- **待用户确认后写入**：{冲突/待确认项}
```

## 使用说明

- 报告在对话中完整呈现，不写入磁盘。
- maintainer 接收后，按"五、写入计划"逐文件写入；冲突项按 `（未证实）` 处理，必要时先向用户确认。
- 一份源文档对应一份报告；跨端/跨模块文档拆分多份。
- 所有 `{...}` 占位在最终报告中替换为实际内容或 `待补充`，不得保留裸占位。
