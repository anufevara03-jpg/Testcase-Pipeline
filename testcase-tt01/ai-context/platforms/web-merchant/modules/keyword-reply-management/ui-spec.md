# 关键词回复管理 UI 交互说明

> 来源：关键词回复管理.md §一、§二、§三、§四、§五

## 页面列表

- 关键词回复管理列表页（路径待补充）

## 页面布局

- 顶部指标区：关键词数 / 启用关键词 / 禁用关键词 / 分组数量（源：§一.1）
- 平台筛选区：按社交平台维护，展示平台下关键词回复数量（源：§二）
- 左侧：话术分组导航（支持分组名模糊搜索、新建/编辑/删除分组）（源：§三）
- 筛选区：关键词（模糊）、发送设置（下拉：全部/原文发送/翻译后发送）、匹配规则（下拉：全部/全匹配/半匹配）、激活码备注（级联多选）（源：§四.1）
- 列表列：关键词 / 适用范围（悬浮激活码明细）/ 匹配方式 / 回复内容（素材摘要）/ 发送设置 / 回复方式 / 延时间隔 / 状态 / 分页勾选（源：§四.2）

## 页面跳转逻辑

- 点"添加关键词回复"/编辑 → 打开新增/编辑关键词回复弹窗（源：§五.1）
- 平台切换 → 刷新平台下话术分组与规则列表；PC 端-关键词回复-预览按平台和激活码适用范围显示（源：§二）
- 操作：编辑、删除、批量删除；删除前二次确认（源：§四.2、§五.2）

## 前端校验规则

| 字段 | 规则 | 触发时机 | 错误提示 | 来源 |
| ---- | ---- | -------- | -------- | ---- |
| 分组名称 | 必填，≤20 字符，商户下不重复 | 提交 | "请输入分组名称" / "分组名称已存在" | §三.1 |
| 关键词 | 必填，≤20 字符，可重复 | 提交 | "请输入/请选择xxx" | §五.1 |
| 话术分组 | 必填，默认当前分组 | 提交 | "请输入/请选择xxx" | §五.1 |
| 匹配规则 | 必填，全匹配/半匹配 | 提交 | "请输入/请选择xxx" | §五.1 |
| 发送设置 | 必填，原文/翻译后发送 | 提交 | "请输入/请选择xxx" | §五.1 |
| 延时间隔 | 必填，范围滑块，默认 2s-5s，自定义 1s-180s | 提交 | "请输入/请选择xxx" | §五.1 |
| 回复方式 | 必填，回复全部/随机一条 | 提交 | "请输入/请选择xxx" | §五.1 |
| 适用范围 | 必填，全部/部分激活码；部分至少 1 个 | 提交 | "请选择适用激活码" | §五.1 |
| 自动回复内容 | 必填，选素材，≤5 条 | 提交/选超限 | "请输入/请选择xxx" / "关键词回复内容最多 5 条" | §五.1 |
| 批量删除 | 至少选 1 条 | 点击批量删除 | 按钮禁用 + 悬浮提示"至少选中一项数据" | §五.2 |

## 异常状态展示

- 单条删除：二次确认"确认删除该记录吗？删除后该记录将不再使用和展示所选内容。"；成功后刷新列表，当前页无数据则回到上一页（源：§五.2）
- 批量删除：确认文案同上；成功后刷新列表并清空已选状态（源：§五.2）
- 删除分组：二次确认"确认删除该分组吗？删除后该分组下的内容将会移动到默认分组。"（源：§三.2）
