# customer-profile-management 需求说明

## 用户故事
- 作为商户管理员，我需要配置客户画像字段模板，以便定义客户详情和PC端客户画像面板的展示字段（源：客户画像管理模块.md §一）

## 验收标准
- [ ] 新增字段后，后台客户详情和PC客户画像面板需立即展示该字段，历史客户显示空值（源：客户画像管理模块.md §二）
- [ ] 编辑字段（名称、选项、分组）后，两端表单同步更新，已保存值保留（源：客户画像管理模块.md §二）
- [ ] 删除/隐藏字段后，两端不展示但历史值保留（源：客户画像管理模块.md §二）
- [ ] 分组排序变更后，后台和PC端展示顺序同步更新（源：客户画像管理模块.md §一§3）

## 业务规则
1. 系统默认分组（联系人信息、销售意向、公司信息）及其字段不可删除、编辑、隐藏，仅可排序（源：客户画像管理模块.md §一§3）
2. 自定义分组名称最长30字符，自定义分组间名称不能重复（源：客户画像管理模块.md §三§1）
3. 自定义字段名称最长10字符，自定义字段间名称不能重复（源：客户画像管理模块.md §四§1）
4. 删除分组时，若分组下存在字段则禁止删除（源：客户画像管理模块.md §三§2）
5. 字段类型支持：文本、多行文本、日期、单选、下拉单选、下拉多选（源：客户画像管理模块.md §四§2）

## 约束与依赖
- 依赖后台客户详情模块展示配置的字段
- 依赖PC端客户画像面板展示配置的字段
