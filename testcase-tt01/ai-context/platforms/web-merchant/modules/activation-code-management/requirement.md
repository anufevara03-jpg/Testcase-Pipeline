# 激活码管理 需求说明

> 端：web-merchant ｜ 模块：activation-code-management
> 主要来源：`TemporaryFile/商户端模块说明/激活码管理模块.md`（下称"模块文档"）

## 用户故事

- 作为商家主账号/员工，我希望按分组管理激活码，以便分配给客服子账号登录 PC 客户端使用。
- 作为商家，我希望配置激活码的端口分配、去重范围、工作时间等，以便控制 PC 端登录与线索去重行为。
- 作为商家，我希望启用/停用激活码并查看登录日志，以便排查 PC 登录与端口占用问题。

## 验收标准

- [ ] 场景：查看统计卡片，预期：展示"激活码总数/启用激活码数/停用激活码数/激活码分组数"四个指标，分组数默认分组计入（源：模块文档 §一、§二.1）。
- [ ] 场景：新建激活码成功，预期：系统自动生成 12 位随机码（大小写字母+数字），同商户下唯一（源：模块文档 §二.2、§五.1）。
- [ ] 场景：必填项为空保存，预期：提示"请输入xxx，请选择xxx"，不关闭抽屉（源：模块文档 §五.1）。
- [ ] 场景：分组名称重复，预期：提示"分组名称已存在"；分组名称最长 20 字符（源：模块文档 §三.2）。
- [ ] 场景：批量操作未勾选，预期：按钮禁用，悬浮提示"至少选中一项数据"（源：模块文档 §五.3、§六.2）。
- [ ] 场景：停用激活码，预期：PC 不可新登录，已登录被强制下线提示"该激活码已停用或删除"，回收端口，对应工单置禁用（源：模块文档 §五.2、§六.2）。
- [ ] 场景：多设备登录关闭时新设备登录，预期：旧设备被挤下线，提示"此激活码已在新设备登录"（源：模块文档 §四.8、§五.6）。
- [ ] 场景：端口不足，预期：提示"当前端口数已达上限，请提升端口数或升级套餐"（源：模块文档 §五.6）。

## 业务规则

1. 激活码由系统自动生成，12 位随机码（大小写字母+数字），同商户下唯一（源：模块文档 §二.2、§五.1）。
2. 默认分组为系统分组，不可删除；删除其他分组时其下激活码移入默认分组（源：模块文档 §三.1）。
3. 删除分组不删除激活码；去重范围为"激活码分组内"时，后续线索按移动后分组去重，历史不回溯（源：模块文档 §三.3）。
4. 线索去重范围 6 种：当前工单 / 当前工单(含底库) / 账号下全部工单(含底库) / 关联账号下全部工单(含底库) / 激活码分组内 / 指定激活码去重；默认"当前工单"（源：模块文档 §四.5）。去重仅影响非首次登录进入的线索，不影响首次上报/手动上报/线索清零/工单重复；仅同平台内去重，跨平台隔离（源：模块文档 §二.3、§二.4）。
5. 端口分配：动态分配（默认）/ 固定分配（>0）；动态分配可用数 = 端口总数 − 固定分配总数 − 动态分配类型下在线社交账号数（源：模块文档 §四.1）。
6. 启用→可用于 PC 登录；停用→PC 不可新登录、已登录强制下线、回收端口、对应工单置禁用（源：模块文档 §五.2）。
7. 社交平台多选必填（WhatsApp/Telegram/Line），默认全选；PC 登录后只展示允许登录所选平台（源：模块文档 §四.4）。
8. PC 登录前置校验：激活码存在、状态启用、平台已配置、端口有余额、当前时间符合工作时间（源：模块文档 §五.6）。
