# 数据指标查看 需求说明

> 端：web-admin ｜ 模块：data-metrics ｜ 对应：聚合 A/B 域表只读统计

## 用户故事

- 作为平台运营，我希望查看商家维度的数据指标（商家数、活跃商家、套餐分布、到期预警），以便掌握平台商家经营状况。
- 作为平台运营，我希望查看商业化指标（套餐销售/付款记录/购买账号发货/余额），以便监控营收。
- 作为平台运营，我希望查看资源消耗指标（翻译字符消耗、翻译次数、端口占用、消息量），以便掌握资源使用与计费健康度。
- 作为平台运营，我希望查看平台接入指标（各社交平台接入分布、各翻译线路调用量与成功率），以便评估接入质量。

## 验收标准

- [ ] 场景：商家指标看板，预期：展示商家总数、按套餐/状态分组统计、到期预警列表（基于 `zs_scrm_merchant_setmeal.end_time`）。
- [ ] 场景：商业化指标，预期：按时间维度统计 `zs_scrm_setmeal_order` 的订单量/金额（区分 order_category 充值套餐 vs 购买账号）、支付状态分布、发货状态分布。
- [ ] 场景：翻译消耗指标，预期：聚合 `zs_scrm_translate_record` 统计字符数、费用、按线路/商家/时段分布、翻译成功率（status）。
- [ ] 场景：端口占用指标，预期：基于 `zs_scrm_device_port` 统计当前占用会话数（occupy_state=1）、按商家分布、套餐端口利用率。
- [ ] 场景：消息量指标，预期：聚合 `zs_scrm_customer_message` 统计消息量、按平台/方向分布（受 chat_backup 开关影响，需注明统计口径）。
- [ ] 场景：平台/线路接入指标，预期：各 `zs_scrm_social_platform` 绑定账号数分布、各 `zs_scrm_translate_line` 调用量与成功率。
- [ ] 场景：指标筛选，预期：支持时间范围、商家、套餐、平台、线路等维度筛选。

## 业务规则

1. 数据指标为只读聚合视图，不修改业务数据。
2. 跨租户统计（平台全局视角），不受单一 `merchant_id` 隔离限制，但需平台管理员权限。
3. 消息量统计受激活码 `chat_backup` 开关影响（关闭时不落正文，规则 #11），统计口径须注明。
4. 翻译消耗指标以 `translate_record.cost` 为计费依据（规则 #7）。
5. 端口占用以 `device_port.occupy_state=1` 为准（规则 #3/#5）。

## 约束与依赖

- 依赖 A 域配置表与 B/C 域业务表的只读聚合（大表 message/translate_record 建议按月分表/归档，统计需考虑分表）。
- 指标定义与口径需与产品确认；预警阈值（如到期预警天数）待补充。
- 接口/数据模型/UI/测试笔记：暂未提供，待补充。
