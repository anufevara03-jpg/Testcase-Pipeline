# pc-client 端概览

## 端简介

- **端名称**：pc-client
- **端类型**：PC 客户端（Electron 桌面应用，版本 1.0.7）
- **目标用户**：商家分发的激活码登录端（商家员工 / 子账号通过激活码登录使用）

> 说明：作为 SaaS 后台的客户端载体，支持子账号登录与多平台聚合。集"环境隔离（防关联）"与"智能翻译沟通"于一体的海外营销客户端。

## 核心定位

1. **防关联与多账号矩阵**：通过独立代理环境 + 指纹隔离 + Cookie 管理，为每个账号提供独立浏览器/网络指纹环境，规避账号关联封禁。
2. **跨境沟通语言壁垒**：内置聚合翻译引擎（GPT-4o-mini 等 AI 线路 + 传统翻译），提供截图翻译、语音翻译。
3. **团队协作与权限分层**：主账号在用户后台创建子账号并分配不同平台权限。
4. **稳定的全球网络基建**：内置全球专线与安全加密链路，保障低延迟（UI 显示 905ms）与高可用。

## 技术栈

- **前端框架**：Electron
- **核心引擎**：聚合翻译引擎（GPT-4o-mini / 自定义提示语）、消息引擎、多平台适配器
- **隔离层**：代理管理 + 指纹与浏览器环境（UA/WebGL/随机指纹/Cookie）

## 架构分层

接入与通讯层（全球专线 / 子账号认证）→ 环境隔离层（代理 / 指纹，**所有业务的前置依赖**）→ 业务聚合层（多平台适配器 / 消息引擎 / 聚合翻译 / 客户管理 / 营销自动化）→ 配置与监控层（全局配置 / 用量监控 / 客服反馈）。

## 核心模块职责

| 模块 | 核心职责 | 关键依赖/事务边界 |
| ---- | -------- | ----------------- |
| 子账号认证模块 | 校验登录权限、拉取被授权平台列表 | 依赖用户后台主账号配置；仅鉴权，不负责社媒登录状态 |
| 代理管理模块 | 全局代理 / 单会话独立代理（HTTP/Socks），智能填写与检测 | 所有平台适配器启动时必须调用获取网络出口 |
| 指纹与浏览器环境模块 | 生成独立浏览器指纹、Cookie 隔离 | 多平台适配器初始化 Web 容器时注入；退出即销毁 |
| 聚合翻译引擎 | 统一 AI 提示语、线路切换、源/目标语言、截图翻译、独立翻译开关 | 消息引擎收发时调用；不存储翻译记录（除字符优化） |
| 多平台适配器 | 对接各平台底层协议，代理+指纹发起连接 | 只做协议适配；群发限频等由上层控制 |
| 营销自动化模块 | 快捷回复预设、群发消息任务 | 群发任务具事务性（成功/失败记录），不阻塞其他会话 |
| 用量监控模块 | 实时显示剩余字符数 / 翻译次数 | 翻译与消息发送实时回调扣减；只读+本地缓存，余额以云端为准 |

## 端到端业务流（完整故事线）

启动客户端 → 子账号登录 → 用户后台权限校验（无权限提示前往后台创建子账号）→ 加载全局配置（专线/代理/指纹）→ 初始化隔离环境 → 主界面展示已授权平台 → 选择平台打开 → Cookie/会话有效性校验（无效则代理环境自动登录）→ 实时同步平台消息与粉丝互动 → 接收客户消息 → 判断是否需翻译（是→聚合翻译引擎）→ 多语言回复发送 → 客户备注与标签管理（粉丝洞察/分类）→ 运营动作（手动跟进 / 自动化营销 / 批量群发）→ 数据统计与用量监控 → 系统设置维护（开机启动/消息通知/主题）。

异常：链路异常依赖全球专线重连确保加密链路稳定。

## 与其他端的关系

- 与 web-admin 共享：翻译线路/计费配置（[[common-business-rules]] 规则 #7/#8）、三方平台字典、套餐端口额度。
- 与 web-merchant 共享：激活码配置（规则 #2）、客户/消息/运营数据（B 域表）、端口占用（规则 #3~#5）。
- 跨端流程：套餐购买 → 激活码生成（web-merchant）→ 客户端登录（pc-client）→ 产生翻译计费（闭环见规则 #7）。

## 约定与备注

- 端口占用按"会话"计：开会话即占端口（空会话也占），关会话才释放（规则 #3）。
- 客户端写入的会话、消息、翻译记录落 B 域/C 域表，受激活码配置开关控制（如 chat_backup，规则 #11）。
